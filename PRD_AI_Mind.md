# Tài liệu Yêu cầu Sản phẩm (PRD) - AI MIND

## 1. Tổng quan dự án

- **Tên dự án:** AI MIND - Dịch vụ đào tạo, tư vấn và triển khai AI GMP/ISO cho nhà máy dược
- **Ngày cập nhật:** 2026-04-18
- **Trạng thái:** Bản nháp v1
- **Loại sản phẩm:** Website dịch vụ B2B + hệ thống thu lead / đặt lịch tư vấn
- **Mục tiêu thay thế:** PRD này thay thế định hướng cũ kiểu "tư vấn khóa học online AI / Son Skool" bằng mô hình dịch vụ đào tạo, tư vấn và triển khai AI GMP/ISO cho doanh nghiệp dược.

### Vấn đề cần giải quyết

Doanh nghiệp dược đang chịu đồng thời 4 áp lực lớn, đúng theo narrative từ ảnh tham chiếu:

- Hồ sơ GMP/ISO quá nhiều, quy trình ký duyệt và kiểm soát còn thủ công.
- Mỗi kỳ thanh tra đều tạo áp lực lớn vì thiếu trạng thái "audit-ready" theo thời gian thực.
- Quy trình thủ công dễ sai sót, chậm phản hồi và phụ thuộc mạnh vào con người.
- Doanh nghiệp muốn số hóa nhưng không biết nên bắt đầu từ GMP, ISO, AI hay Automation trước.

### Giải pháp đề xuất

Xây dựng website và CMS cho AI MIND với mục tiêu:

- Định vị AI MIND là đối tác **đào tạo + tư vấn + triển khai AI GMP/ISO** cho nhà máy dược.
- Làm rõ hai năng lực cốt lõi:
  - **Tư vấn triển khai GMP/ISO kết hợp AI**
  - **Xây dựng hệ thống AI & Automation nhà máy Dược**
- Tạo funnel chuyển đổi từ người xem thành:
  - lead tiềm năng,
  - lịch tư vấn,
  - cơ hội đánh giá hiện trạng,
  - đề xuất dự án triển khai thực tế.

### Tác động kinh doanh

- Giảm sự mơ hồ trong thông điệp: từ "học AI online" sang "giải quyết bài toán GMP/ISO thực tế của nhà máy dược".
- Tăng tỷ lệ lead chất lượng vì website lọc đúng chân dung khách hàng B2B.
- Tăng khả năng chốt hợp đồng tư vấn/đào tạo/automation giá trị cao.
- Tạo nền tảng mở rộng SEO dài hạn quanh các chủ đề GMP, ISO, AI compliance và số hóa vận hành dược.

### Chỉ số thành công

| Chỉ số | Mục tiêu |
|---|---|
| CTA click rate trên landing page | >= 6% |
| Tỷ lệ gửi form / đặt lịch | >= 3.5% |
| Lead được phản hồi trong 2 giờ làm việc | >= 90% |

### 10.1. Thông tin project hiện tại

| Production URL | `https://ai-mind-nguyentuanson167-9894s-projects.vercel.app` |

### 10.2. Setup Firebase từ đầu

1. **Tạo Firebase project** tại https://console.firebase.google.com
2. **Enable 3 dịch vụ:**
   - Firestore Database → Create database → location `asia-southeast1` → Production mode
   - Authentication → Get started → Sign-in method → Email/Password → Enable
   - Storage → Get started → Production mode
3. **Lấy config:** Project Settings → Your apps → Web app → Register → Copy `firebaseConfig`
4. **Tạo `.env.local`** từ `.env.local.example`, điền config values
5. **Cập nhật `.firebaserc`** với đúng project ID

### 10.3. Deploy Firebase Rules & Seed Data

```bash
# 1. Login Firebase CLI (chạy trong terminal riêng, cần mở trình duyệt)
firebase login

# 2. Deploy rules + indexes
firebase deploy --only firestore:rules,firestore:indexes,storage

# 3. Seed data (cần tạm mở rules cho authenticated write, xem Agent.md §12.3)
node scripts/seed-firestore.mjs

# 4. Restore production rules rồi deploy lại
firebase deploy --only firestore:rules
```

### 10.4. Deploy Vercel

```bash
# 1. Thêm env vars (chạy từ thư mục ai-mind/)
echo "value" | npx vercel env add NEXT_PUBLIC_FIREBASE_API_KEY production
# ... lặp lại cho 7 biến

# 2. Set root directory nếu monorepo (qua Dashboard hoặc API, xem Agent.md §12.4)

# 3. Push code → Vercel auto-build
git push origin main
```

### 10.5. Các lỗi thường gặp và cách xử lý

| Lỗi | Nguyên nhân | Cách sửa |
|---|---|---|
| `auth/invalid-api-key` khi build | Firebase init ở module level, build time không có env | Dùng lazy initialization (Agent.md §12.1) |
| `PERMISSION_DENIED` khi seed | Firestore rules chặn write, hoặc user chưa authenticated | Tạm mở rules + sign in lại nếu user đã tồn tại (Agent.md §12.3) |
| `Cannot run login in non-interactive mode` | `firebase login` cần trình duyệt | Chạy trong terminal riêng, không qua CI/Claude Code |
| Vercel build fail 0s "Unexpected error" | Root Directory sai, Vercel không tìm thấy `package.json` | Set Root Directory = `ai-mind` trong Vercel settings (Agent.md §12.4) |
| `The provided path does not exist` khi CLI deploy | CLI đang ở subfolder + root directory đã set | Dùng git push auto-deploy thay vì CLI từ subfolder |
